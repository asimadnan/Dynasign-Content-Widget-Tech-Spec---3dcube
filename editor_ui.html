Build Widget Editor UI JSON File

Editor UI will be generated automatically based on the editor ui JSON file

Editor UI:

The widget editor includes the left side (editable fields) and right side (edit view) as illustrated below. The left side includes a number of tabs. Template editable fields are grouped inside the tabs.
  • Tab (Accordion)
  • Group (Expand/Collapse)
  • Template Fields

Editor UI JSON File (ui_[template name].json)

Add ui JSON file under widget_editor/[template name]/. To build the widget editor, you need to add at least one custom tab to include template specific fields. To start, you can copy ui_annoucement.json and rename it to ui_[template name].json, then edit for your widget temmplate.

    "template":"announcement",
    "standard_tabs":[
            {"tab_id":"background","open":"true","overlay":true},
            { "tab_id":"qr"}
        ],
    "custom_tabs":[
        {
            "tab_id":"text_message",
            "tab_name":"Text Message",
            "groups":[
                {
                    "group_id":"grp_title",
                    "group_name":"Title",
                    "open":true,
                    "fields":[
                        {"field_id":"title","field_name":"Title","field_type":"text","component":"textarea","field_width":"400","field_height":"50",
                        "placeholder":"Enter title","font_type":"font_type_list","font_size":"font_size_list","color":"std",
                        "bold":"std","italic":"std","underline":"std"}
                            ]
                },
                {
                    "group_id":"grp_description",
                    "group_name":"Description",
                    "open":true,
                    "fields":[
                        {"field_id":"description","field_name":"Description","field_type":"text","component":"textarea","field_width":"400","field_height":"100",
                        "placeholder":"Enter description","font_type":"font_type_list","font_size":"font_size_list","color":"std",
                        "bold":"std","italic":"std","underline":"std"}
                            ]
                }
                        ]
            }
                    ],
    "val_lists":{
        "alpha_list": ["0|0%","0.1|10%","0.2|20%","0.3|30%","0.4|40%","0.5|50%","0.6|60%","0.7|70%","0.8|80%","0.9|90%","1|100%"],
        "font_type_list": ["Arial","Calibri","Times New Roman","Verdana"],

        }

Define Template Name (template):

Template name should be all lower cases, no space and no special characters, '_' is allowed, e.g. announcement, twitter, twitter_pic, countdown

Define Standard Tab (standard_tabs):

There are 2 optional standard tabs. Add the following lines to include the tabs in the editor.
  • Background: {"tab_id":"background","overlay":true} The optional overlay on the background can be set to either true or false
  • QR Code: {"tab_id":"qr"}

Define Custom Tabs (custom_tabs):

Editor has one or more custom tabs to contain template fields. custom_tabs is a JSON array.
  • Tab ID (tab_id): tab_id should be all lower cases, no space and no special characters, e.g. text_message
  • Tab Name (tab_name): tab name will show up in the tab,e.g. Text Messages

Define Groups (groups):

groups is a JSON array inside a given custom tab.
  • Group ID (group_id): group_id should be all lower cases, no space and no special characters, '_' is allowed, e.g. title
  • Group Name (group_name): group name will show up in the tab,e.g. Title
  • Open or Close (open): "true": open the group by default

Define Template Field Object (fields):

  • Field ID (field_id): field_id should be all lower cases, no space and no special characters. '_' is allowed
  • Field Display Name (field_name): make it as short as possible
  • Field Content Type (field_type): text, image, video, audio, shape
  • Field Component (field_component): name of the HTML component, see "HTML Components" for all supported components
  • Field Default Value (default)
  • Field Width (field_width) in pixel
  • Field Height (field_height) in pixel
In addition to the above common attributes, each component may have its own specific attributes
  • Field Value List (val_list)

    • HTML Components:

    • text_style
      • font_type
      • font_size
      • color
      • bold
      • italic
      • align
      • underline (not supported yet)
      • shadow (not supported yet)
      • background_color (not supported yet)
      • background_opacity (not supported yet)
      • border (not supported yet)
      • rotate (not supported yet)
    • input (text_input)
      • placeholder
      • maxlength
      • all attributes in text_style can be included as options
    • textarea
      • placeholder
      • all attributes in text_style can be included as options
    • checkbox
    • radioRadio button
      • val_list: a list of selection values
    • selectSelect Dropdown
      • val_list: a list of radio button values
    • comboCombo Dropdown
      • val_list: a list of selection values
    • overlay (overlay_rectangle)
    • color: Color Picker
    • date: Date Picker
    • time: Time Picker
    • dropzoneFile Upload: to simulate file upload from your local drive in this kit, please copy your image files into /ds_widget/img/ folder
    • button_on_off On/Off button
    • number_plus_minusNumber Selector
    • rangeRange Slider
    • label
    • hidden

    Value Lists (val_lists):

    include an array of coded values and display values (coded value | display value). If the coded values are the same as the display values, include the coded values only. Examples:
    • "alpha_list": ["0|0%","0.1|10%","0.2|20%","0.3|30%","0.4|40%","0.5|50%","0.6|60%","0.7|70%","0.8|80%","0.9|90%","1|100%"]
    • "font_type_list": ["Arial","Calibri","Times New Roman","Verdana"]
