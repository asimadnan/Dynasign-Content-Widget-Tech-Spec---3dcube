About this Kit

Dynasign Content Widget Development Kit allows you to develop Dynasign Content Widgets on your computer (localhost) without connecting to Dynasign Studios backend servers. It provides a local environment for you to develop and test widget templates on your computer.

  • Widget Template: a content presentation template with a number of programmable parameters such as text, color, image, style, or external data feeds.
  • Widget Editor: a content programming user interface for end users to program and publish a widget for a given widget template.
  • Widget Player: a testing utility that plays the widget published from Widget Editor.

This development kit includes the basic framework for you to develop and test both a widget template and its editor. The kit also includes a few sample widgets that you can use as a reference for building your new widgets.

Setup

To use this kit for development work, you need to setup a local Web server on your working computer. Any Web Server is fine. Apache HTTP Server is recommended.

  1. Download ds_widget.zip .
  2. Unzip ds_widget.zip. Copy the ds_widget folder into a location where you can map to your local Web server, e.g. c:\mywork\ds_widget
  3. Map your local Web server so you can access the "ds_widget" folder via localhost in your Web Browser (Chrome is preferred), e.g. by entering http://localhost/ds_widget/ in your Web browser, you should see the content of this page.
  4. This kit includes a few sample widget templates and widget editors, plus the basic library files you need to use in your development.

Content Widget Development Process

  1. Develop ideas and visual design
  2. Develop technical specification (template variables, objects, animation, layout, size and position for different zones)
  3. Develop widget editor (using widget development kit)
  4. Build and test widget template (using widget development kit and relevant Javascript libraries)
  5. Submit your widget code to support@dynasign.net.
    • Zip the entire ds_widget folder with both your template files and editor files in it.
    • Rename the zip file to something like ds_widget_[your template name].zip.rename to avoid rejection by email services.
    • In the email subject line, enter "Dynasign Content Widget - [your widget name]", attach the renamed zip file and send the email to support@dynasign.net.

A Sample Widget "announcement"

By reviewing those widget files, you will understand all the building blocks for a widget.

  • Widget Technical Spec: spec_announcement.html
  • Widget Editor Files: widget_editor.html?template=announcement
    • Main editor file: use this file to start the editor, no need to change this file widget_editor.html
    • Template field file(left template fields): widget_editor/announcement/editor_announcement.html
    • Template view file (right side edit view): widget_editor/announcement/view_announcement.html
    • Template ui file: editor UI definition for auto generation widget_editor/announcement/ui_announcement.json
    • Template test data file: test data used by both editor and player widget_editor/announcement/data_announcement.json
    • Template def file (not directly used by the kit): template field default values widget_editor/announcement/def_announcement.json
  • Widget Template Files:
    • Template css folder: htmlplayer/css
    • Template js folder: htmlplayer/js
    • Template folder: htmlplayer/templates
    • Template HTML file: htmlplayer/templates/announcement/template_announcement.html
    • Template JS file: htmlplayer/templates/announcement/template_announcement.js
    • Template Base JS file: to be inherited by all template JS files. No need to change this file htmlplayer/templates/template_base.js
    • Template Appcache Manifest file: htmlplayer/templates/announcement/template_announcement.appcache
  • Player: use this file to play the widget, no need to change this file widget_player.html?template=announcement

How to Build Widget Editor

The widget editor user interface is generated automatically based on the editor ui json file. Please follow the steps below to build your widget editor.

  1. Use the "announcement" editor files for your reference to create your own editor files
  2. Create an editor folder widget_editor/[template name]/ using the template name, e.g. announcement if the folder doesn't exist
  3. Build the 4 editor files using the sample editor files, follow the same naming convention as used in the sample widget editor:
    • Copy editor_announcement.html to the editor folder, rename it to editor_[template name].html.
    • Copy view_announcement.html to the editor folder, rename it to view_[template name].html.
    • Edit view_[template name].html to replace src="../../htmlplayer/templates/announcement/template_announcement.js"> in the script include section to your template js file. You may not have the template js developed yet. That is fine for now.
    • Edit view_[template name].html to replace var m_template = new TemplateAnnouncement(1); with your template name
    • Build editor ui JSON file: ui_[template name].json, refer to Build Widget Editor UI JSON File
    • Build template test data file: Template test data file: data_[template name].json, refer to Build Data JSON File
  4. Launch your editor: enter widget_editor.html?template=[template name] in browser
At this point, you should see all the fields are populated in the left side of the editor. If you haven't had your template code ready yet, your right side edit view will not work. After you finish to code your template, you may continue to the following steps to test your editor and template together.

  1. Test Update: change field values to see if they change in Edit View accordingly.
  2. Test Preview: click the "Preview" button to make sure the field values in the editor will reflect in the popup preview.
  3. Test Save: click the "Save" button to make sure all the field values in the editor are part of Content JSON to be sent.
  4. Test all browsers: test your editor in Chrome, Edge, IE, Firefox, and Safari.

How to Build Widget Template

Please follow the steps below to develop your template

  1. Define a system name for your new template, all lower cases, using _ to connect 2 words, e.g. announcement, countdown
  2. Create a new folder under htmlplayer/templates/ using the template name, e.g. announcement
  3. Create template files using the sample template files, follow the naming convention below:
    • Template HTML file: htmlplayer/templates/[template name]/template_[template name].html
    • Template JS file: htmlplayer/templates/[template name]/template_[template name].js
    • Template Appcache Manifest file: htmlplayer/templates/[template name]/template_[template name].appcache
    • Make sure template test data file: data_[template name].json is present in the editor folder (created during building the editor)
  4. Develop template JS file (template_[template name].js). This is the core of a template. To develop a template, you can choose one of the following methods for your stage:
    • Pixi with transparent stage
    • Pixi with non-transparent stage (without standard widget background)
    • HTML5 canvas stage
    • HTML5 non-Canvas stage
    Please refer to the sample template for general code flow and structure. Use htmlplayer/templates/sample/template_sample.js for your reference. Template JS should always inherit from template_base.js, which includes the basic methods shared by all templates. In addition, you must develop the following methods in your template_[template name].js:
    • initTemplateFields: initialize all template fields
    • determineObjInfo: calculate object sizes and positions for different zone sizes
    • initTemplateUI: build template UI objects
    • playTemplate: start to play template
    • initEditView: initialize the edit view
    • updateEditView: rebuild UI objects to handle updates from the editor
  5. Develop additional methods needed for your template
  6. Launch your editor widget_editor.html?template=[template name] in browser to test your template (see step 5 - 8 under "Build Widget Editor")
  7. Test your template using Widget Player widget_player.html?template=[template name]
